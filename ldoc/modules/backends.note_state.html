Module backends.note_state

Active MIDI note state backend.

Listens on a TCP socket for incoming note updates from the backend thread, reads a local active_notes.lua file, and merges both sources into a unified list of active MIDI notes. Exposes lookup functions for whether a given circle-of-fourths step is active or is the bass (lowest) note.

tryReloadNotes () Reloads notes from active_notes.lua if available.
rebuildState () Rebuilds merged state from file and socket sources.
